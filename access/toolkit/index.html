stuck? this is usually why

5 things first-timers hit

  • "SKILL.md must start with YAML frontmatter" error on Skills upload. you uploaded the wrong file. ai-builder-setup-v2.zip is the bundle for Project / Claude Code paths · it has 27 sub-folders, and claude.ai's Skills upload doesn't accept that shape. download ai-builder-skills-individual-zips.zip instead (file 2 above) · unzip it · upload each of the 27 small zips one at a time.
  • "i can't find Skills in Settings → Features." you're on the free plan · the Skills feature only shows up on Pro, Max, Team, or Enterprise. use path A (Claude Project) instead · works on every plan including free.
  • "claude isn't using my brain doc." the brain doc lives in Project knowledge (path A) or as a chat attachment (path B). if you just pasted it in one chat and then started a new chat, the new chat doesn't see it. either re-attach, or move it to a Project.
  • "the upload says 'too many files'." claude.ai caps project-knowledge file count. upload in batches of ~10 · the bundle has 27 skill folders so split into 3 batches if needed.
  • "claude isn't activating the right skill." be specific in your ask. "build me something for my business" is too vague. "build me a homepage" or "build me a pricing page" is what triggers the right one. (the skill name in the YAML description is what claude matches against your wording.)
  • "i unzipped but i don't see the files." on mac, look for a new folder called ai-builder-setup in the same place where the zip downloaded (usually Downloads). on windows, the extracted folder appears next to the zip.
