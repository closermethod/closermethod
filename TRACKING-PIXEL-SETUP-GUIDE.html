UTM Naming Conventions (Best Practices)

  • Lowercase: Always use lowercase for consistency (utm_source=instagram, not utm_source=Instagram)
  • Hyphens for Spaces: Use hyphens instead of spaces (utm_campaign=spring-launch, not spring launch)
  • Be Descriptive: Use clear names that describe the specific campaign (utm_campaign=authenticity-challenge, not utm_campaign=promo)
  • Consistency: Use the same utm_source, utm_medium values across all campaigns for proper grouping
  • Avoid Duplicates: Different URLs with same UTMs = same campaign in analytics

Using a UTM Builder

To simplify UTM creation, use Google's Campaign URL Builder:

  1. Enter your website URL
  2. Fill in Campaign Source, Medium, Name, Content, Term
  3. Copy the generated URL
  4. Use in your social posts, emails, ads

How GA4 Tracks UTM Parameters

GA4 automatically captures UTM parameters from your URLs and organizes them in reports under:

  • Acquisition: Reports → User Acquisition (utm_source, utm_medium, utm_campaign)
  • Traffic Source: Source/Medium breakdown
  • Campaign: Campaign performance metrics
  • Custom Reports: Create detailed UTM analysis

Kit Commerce Webhook for Server-Side Tracking

Track purchases directly from Kit Commerce to ensure accuracy. Configure a webhook in Kit's settings:

Kit Commerce Webhook Configuration
{ "webhook_url": "https://yourdomain.com/api/track-purchase", "events": ["purchase.completed"], "payload": { "product_id": "product_id", "customer_email": "email", "amount": "total_price", "transaction_id": "order_id" } }

Then create an endpoint on your server to handle this webhook and send data to Meta Pixel and GA4:

Server-Side Webhook Handler (Node.js Example)
// Node.js / Express endpoint to handle Kit Commerce webhook app.post('/api/track-purchase', async (req, res) => { const { product_id, customer_email, amount, transaction_id } = req.body; // Send to Meta Pixel (server-side) const pixelPayload = { data: [{ event_name: 'Purchase', event_id: transaction_id, event_time: Math.floor(Date.now() / 1000), user_data: { em: customer_email // Email hash }, custom_data: { value: amount, currency: 'USD', content_type: 'product', content_ids: [product_id] } }] }; // POST to Meta Conversions API // (requires API token from Meta) // Send to GA4 (via Measurement Protocol) const ga4Payload = { api_secret: 'YOUR_GA4_API_SECRET', measurement_id: 'G-XXXXXXXXXX', client_id: transaction_id, events: [{ name: 'purchase', params: { currency: 'USD', value: amount, transaction_id: transaction_id, items: [{ item_id: product_id, price: amount, quantity: 1 }] } }] }; res.json({ success: true }); });

Step 4: Set Up Triggers for Events

Create triggers that fire tags based on user actions:

  • All Pages trigger: Fires on every page view
  • Product page trigger: Page path contains product URL
  • Click trigger: When user clicks checkout button
  • Custom event trigger: When custom event fires
  • Scroll trigger: When user scrolls 25%, 50%, 75% down page

Step 5: Test GTM Setup

  1. In GTM, click "Preview"
  2. Enter your website URL
  3. Visit your site - GTM preview shows which tags fired
  4. Navigate through your funnel and verify events
  5. When satisfied, click "Submit" to publish

Why Use GTM?

  • Centralized Management: All tracking code in one place
  • No Code Editing: Update tags without touching HTML
  • Version Control: Revert to previous versions if needed
  • Faster Implementation: Deploy new tracking in minutes
  • Built-in QA: Preview mode for testing before publishing
  • Team Collaboration: Non-developers can manage tags
Decision: Use GTM if you plan frequent tracking updates. For simpler implementations, direct code installation is fine.

6. Retargeting Audiences to Create

Build retargeting audiences to serve ads to users based on their behavior in your funnel. These audiences help you re-engage interested prospects.

In Meta Ads Manager

A. All Website Visitors (180 Days)

Purpose: Retarget anyone who visited any page on your site

  1. Go to Ads Manager → Audiences
  2. Create New → Custom Audience → Website Traffic
  3. Pixel: Select your Meta Pixel
  4. Include: All website visitors
  5. Time period: Last 180 days
  6. Name: "Website Visitors - 180 Days"
  7. Create audience

B. Tool Users (Non-Buyers)

Purpose: Users who tried a free tool but didn't purchase

  1. Create New Custom Audience → Website Traffic
  2. Include: People who triggered "ToolUsed" event
  3. Exclude: People who triggered "Purchase" event
  4. Time period: Last 90 days
  5. Name: "Tool Users - No Purchase"
  6. Create audience

C. Product Page Visitors (Non-Buyers)

Purpose: Users who viewed a product page but didn't buy

  1. Create New Custom Audience → Website Traffic
  2. Include: People who triggered "ViewContent" event
  3. Exclude: People who triggered "Purchase" event
  4. Time period: Last 60 days
  5. Name: "Product Viewers - No Purchase"
  6. Create audience

D. Cart Abandoners

Purpose: Users who started checkout but didn't purchase

  1. Create New Custom Audience → Website Traffic
  2. Include: People who triggered "InitiateCheckout" event
  3. Exclude: People who triggered "Purchase" event
  4. Time period: Last 30 days
  5. Name: "Cart Abandoners"
  6. Create audience

E. Buyers (For Lookalikes)

Purpose: Users who purchased - use to find similar people

  1. Create New Custom Audience → Website Traffic
  2. Include: People who triggered "Purchase" event
  3. Time period: Last 180 days (all buyers)
  4. Name: "All Buyers"
  5. Create audience

Then create a Lookalike Audience:

  1. Create New Audience → Lookalike Audience
  2. Source: "All Buyers"
  3. Location: United States (or your target country)
  4. Audience size: 1% (most similar to buyers)
  5. Name: "Lookalike - Buyers (1%)"
  6. Create audience
